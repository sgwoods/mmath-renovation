# Historical Validation Matrix

This matrix maps the historically important AbTweak experiment themes onto the recovered `Abtweak-1993` working tree.

It is intended to answer two questions:

1. what should we run to claim the 1993 system is restored?
2. how do we compare our modern runs with the results described in the papers, thesis, and manual?

## Extracted Historical Expectations

The main source-backed expectations we should validate against are:

1. AbTweak with the monotonic property should usually beat breadth-first abstract planning without monotonic pruning on good hierarchies, while poor hierarchies can erase or reverse that advantage. Sources: [AAAI 1990](https://cdn.aaai.org/AAAI/1990/AAAI90-031.pdf), [TR-91-65](https://cs.uwaterloo.ca/research/tr/1991/65/final.pdf), [Thesis-91-17](https://cs.uwaterloo.ca/research/tr/1991/17/report.pdf)
2. Left-Wedge is historically presented as a completeness-preserving control strategy that can improve search dramatically on good abstraction hierarchies, but can perform worse on bad ones. Sources: [TR-91-65](https://cs.uwaterloo.ca/research/tr/1991/65/final.pdf), [Thesis-91-17](https://cs.uwaterloo.ca/research/tr/1991/17/report.pdf)
3. In the robot task-planning domain, the reported strong result combines abstraction with the primary-effects heuristic, monotonic property, and Left-Wedge; without the heuristic, the report says many simple problems were not solvable by the planners tested. Source: [TR-91-65](https://cs.uwaterloo.ca/research/tr/1991/65/final.pdf)
4. The 1993 manual explicitly treats simple blocks, Nilsson blocks, Hanoi, registers, and robot planning as the core sample domains of the distribution, so successful runs on those domains are the best first restoration targets. Sources: [`working/abtweak-1993/Doc/users-manual.tex`](/Users/stevenwoods/mmath-renovation/working/abtweak-1993/Doc/users-manual.tex#L448), [CMU AI Repository page](https://www.cs.cmu.edu/afs/cs/project/ai-repository/ai/areas/planning/systems/abtweak/0.html)

## Immediate Benchmark Priorities

The next most valuable checks are:

1. keep `blocks` / `sussman` as the first passing `tweak` regression
2. use the source-backed expectations above to annotate each modern benchmark as matching, partially matching, or still untested against the historical claims
3. label the new computer, biology, fly, and database sample cases against the historical claims
4. use both robot domains to compare left-wedge and user-heuristic behavior against the published expectations
5. then widen into additional 1993 sample domains such as scheduling and stylistics

## Comparison Rules

When comparing with the historical documents, prefer this order:

1. same domain and planner mode
2. same qualitative outcome, such as solved versus unsolved
3. same plan shape or plan-size scale
4. same search trend when toggling abstraction or left-wedge
5. same raw counts if the implementation and settings are close enough

If a modern run differs, document whether the most likely cause is:

- an incomplete SBCL port
- a change in search bounds or defaults
- a historically implementation-dependent Lisp behavior
- a real disagreement between the recovered code and the reported result
